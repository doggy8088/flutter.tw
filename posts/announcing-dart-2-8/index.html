Dart 2.8 同期發布

目錄

  • 更優質的 package 生態
  • Dart 2.8 發布工具性能改進
  • 使用 pub outdated 管理依賴

作者 / Michael Thomsen, Dart & Flutter Product Manager, Google

我們同樣為大家帶來了 Dart SDK 的新版本: Dart 2.8。Dart 社群保持著驚人的增長，如今有 數百萬 Flutter 開發者 使用 Dart 作為針對用戶端最佳化的開發語言，在各個平台上建立高速流暢的應用。我們仍在努力完成 即將推出的空安全 (null safety) 功能，使 Dart 成為一種更最佳化的語言，打造高速且穩定的使用者介面。我們準備了一些激動人心的新功能，讓開發者在管理依賴關係時更加高效。

Dart 平台透過 pub 用戶端工具 和 pub.dev package 函式庫內建了程式碼套件管理功能。在過去的一年裡，pub.dev package 函式庫增長了 200%，現在已經擁有近 10,000 個 package。作為持續改進 Dart 生態系統的一環，Dart 2.8 SDK 為 pub 用戶端工具帶來了兩個改進: 更強的 pub get 效能，以及一款新工具，可確保你的 package 依賴始終保持最新。

Dart 2.8 還在 Dart 語言和程式碼庫中引入了一些小幅度的重要改動 (Breaking Changes)。這些變更為我們的第一版空安全功能奠定了基礎。

為空安全奠基

應用崩潰的一個常見原因是程式碼試圖使用一個恰好為空 (null) 的變數。Tony Hoare 爵士於 1965 年在 ALGOL 程式語言中引入了空引用，他在 2009 年 QCon 演講 中，把空引用稱為 “價值十億美元的錯誤”。在某些情況下，空值是有用的；但難點在於如何把這些有用的情況甄別出來。在過去的一年中，我們一直忙於在 Dart 中打造 健全的空安全支援。這種支援將擴充套件型別系統，以表達始終不可空的變數；型別系統也會是完全健全的: Dart 編譯器和執行時都將信任這些型別，並能夠在型別系統保證變數不為空的情況下生成最佳化過的程式碼。

如你所想，這是一項浩大的工程，牽涉到的內容很多。為了確保用簡潔的語義建立空安全，我們決定在 Dart 語言和庫中引入一些小幅的重要改動。這些重要改動是對 Dart 型別系統邊界情況和一些 Dart 核心函式庫所做的小調整，以確保可空性相關的可用性和效能。我們已經在 Dart 公告清單中 預先宣佈 了這些改動，而且我們估計這些重要改動對普通應用的程式碼影響很小。如果你在使用 Dart 2.8 時遇到任何問題，建議你檢視這些 重要改動，瞭解其詳情以及應對方法。如果這樣依然不能解決問題，請在我們的 問題反饋頁 送出報告。想提前體驗空安全，請 透過 DartPad 一睹為快。

更優質的 package 生態

Pub package 管理器和 pub.dev 網站為 Dart 和 Flutter 提供了欣欣向榮的生態系統。在 pub.dev 上有近 10,000 個 package，許多有趣的 package 更是讓我們拍案叫絕。我們的核心任務之一就是幫助眾多作者打造高品質的 package，並幫助開發者找到它們。在這方面我們已經做了一些工作，比如 改進 pub.dev 的可發現性，Verified Publishers (發布者認證)，以及 Flutter Favorite 專案。

現在你從 pub.dev 獲取 package 的速度會快上許多。我們還提供了一款新工具來幫助你更新所有 package 的依賴關係。現在的應用頗為複雜，很可能依賴幾十個 package。如何把它們都更新到最新版本，確保滿足最新的依賴關係，並獲得所有的 bug 修復和效能最佳化？從 Dart 2.8 開始，你可以使用 pub outdated，這款新工具將自動判斷哪些依賴項需要更新到最新、最合適的版本。

Dart 2.8 發布工具性能改進

我們最佳化了 pub 工具的效能，在執行 pub get 時會並行獲取 package，並且推遲了 pub run 預編譯的執行時機。

在使用 flutter create 建立的新專案上執行 flutter pub get 的非正式基準測試，使用 Flutter 1.12 (使用 Dart 2.7) 總執行時間約為 6.5 秒，而在 Flutter 1.17 (使用 Dart 2.8) 中下降至 2.5 秒。在 Flutter Gallery 這樣更大型的應用中，時間則從約 15 秒降到了 3 秒左右！

使用 pub outdated 管理依賴

Dart 程式碼中的依賴關係會被收集到 pubspec 檔案 中。當你執行 pub get 指令，從 pub.dev 中獲取 package 時，pub 版本求解器 (使用 PubGrub 演算法) 會執行一個程序來得出滿足 pubspec 中所有約束條件的所有依賴項的最新版本。請注意，pub 使用的是單版本方案，你的應用中只包含每個 package 的單一版本，這個方案可以確保你的應用獲得儘可能小的體積。

始終使用最新的穩定版 package 是 最佳開發實踐 之一，但這樣做會很費力。Dart 支援使用 pub upgrade 升級到 語義上相容 的最新版本，但如果不更新 pubspec，就不能包含 package 最新的大版本。pub outdated 指令透過比較當前使用的版本和 pub.dev 上的最新版本，讓你知曉小版本和大版本何時可用。

下一步

效能改善、重要改動，以及全新的 pub outdated 指令，現已加入穩定版 Dart 2.8 SDK 和穩定版 Flutter 1.17 SDK。我們建議你儘快開始使用 pub outdated，掌握自己專案中依賴的健康狀況！

如果你遇到了問題，請前往 pub 問題反饋頁 報告給我們；如果你遇到了通用問題，請前往 SDK 問題反饋頁 進行上報。我們非常期待大家分享 pub outdated 的使用體驗。
