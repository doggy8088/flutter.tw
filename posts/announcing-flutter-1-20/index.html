作者 / Chris Sells, Product Manager, Flutter developer experience

我們對 Flutter 的願景是提供一個可移植的工具包，讓你無論在任何螢幕上都能隨心所欲地繪製畫素，打造出美好的體驗。每次更新，我們都著力確保 Flutter 能夠在所有支援的平台上執行流暢、介面美觀、開發高效而且保持開放性。透過穩定版渠道發布的 Flutter 1.20 在上述四個方面都進展頗多。

首先是執行流暢，我們從最底層的渲染引擎到 Dart 語言本身都實現了多項效能改進。

為了讓你建立出更美觀的 Flutter 應用，此版本提供了多項介面改進，包括期待已久的自動填充支援、支援平移和縮放的新 widget 佈局方式、新的滑鼠游標支援、舊版本中人氣 Material widget（如時間和日期選擇器）的更新，以及為桌面和行動端 Flutter 應用中的關於（About）介面帶來了全新的回應式許可（license）展示頁面。

為了進一步提高開發效率，我們更新了 Visual Studio Code 的 Flutter 擴充套件，將 Dart DevTools 直整合進 IDE，在你移動檔案時會幫你自動更新匯入語句，並提供了一組新的元資料用於建立你自己的工具。

得益於 Flutter 的開放性和來自社群的出色貢獻，此版本包含了全球 359 名貢獻者（其中包括來自 Flutter 社群的 270 名貢獻者）的 3,029 個合併 PR，關閉了 5,485 個 issue。因此本次更新的 Flutter 版本也是目前為止擁有最多貢獻者的版本。在這裡特別感謝在社群中貢獻了 28 個 PR的 CareF，貢獻了 26 個 PR 的 AyushBherwani1998（包括他用於 Google Summer of Code 專案的 10 個 Flutter 範例），以及貢獻了 13 個 PR 的 a14n（其中一大部分用於支援 Flutter 的空安全性，有關該主題的更多訊息即將到來！）。Flutter 的誕生離不開社群貢獻者們的廣泛支援。謝謝大家！

Flutter 每一個新版本的發布都伴隨著使用量的增長和更迅猛的發展態勢。事實上，在 4 月份我們曾 報道過 Google Play 商店中 Flutter 應用的數量已經達到 50,000，月度新增應用數量峰值更是高達 10,000。現在，短短三個月後，Google Play 中的 Flutter 應用數量已經超過 90,000。增長最快的當屬印度，那裡已經是 Flutter 開發者最多的地區，開發者數量在過去六個月中翻了一番，這與 Google 在該地區增加的投資 密切相關。最後，如果沒有 Dart 語言，Flutter 也不會成為現在的 Flutter。這裡分享一個好訊息：在 IEEE 的 開發語言榜單 中，Dart 相比去年上升了 4 位，在榜單的前 50 種語言中排名第 12。

Flutter 和 Dart 的效能改進

Flutter 團隊一直在尋找縮減應用大小和延遲的新方法。對於大小，此版本修復了在進行圖示字型搖樹（tree-shaking）操作時的工具性能問題，並在你建立非 web 應用時預設進行 字型搖樹操作。圖示字型搖樹操作會移除應用中未使用的圖示，從而縮減其大小。在對 Flutter Gallery 應用進行該操作後，我們發現 應用大小縮減了 100kb。現在，在建立移動版應用的發布版本時該操作會預設執行。目前僅限於 TrueType 字型，但在未來版本中將取消這個限制。

此版本帶來的另一項效能改進是使用預熱階段減少動畫初始顯示時的卡頓。以下為卡頓改進的動畫範例（半速播放）。

△ 使用和不使用 SkSL 預熱的動畫

如果 Flutter 應用在首次執行時的動畫出現卡頓，那麼 Skia Shading Language 著色器將在應用建立中提供預編譯，將速度提高 2 倍以上。如果你想使用此高階功能，請參見 flutter 文件中的 SkSL 預熱頁面。

最後，在針對桌面環境的最佳化中，我們進一步完善了對滑鼠的支援。在此版本，我們 重構了滑鼠點選測試系統，帶來了許多曾因效能問題受阻的架構優勢。在基於 web 的微型基準測試中，重構使效能提高了多達 15 倍！這意味著，你可以在保證效能的前提下，獲得更好、更一致、更準確的點選測試結果：實現雙贏！

有了更好、更快、更強大的滑鼠點選測試，我們又增加了滑鼠游標支援，這也是桌面端最受期待的功能之一。一些常用的 widget 將預設顯示主流游標，你也可以從支援的游標清單中指定其他游標。

△ 滑鼠在 Android 既有的 widget 上懸停時切換顯示游標

此版本 Flutter 基於 2.9 版 Dart 建立，採用基於狀態的全新雙通 UTF-8 解碼器，解碼原語在 Dart VM 中最佳化，部分利用了 SIMD 指令。UTF-8 是目前網際網路上最常用的字元編碼方法。在接收大型網路回應時，快速對其進行解碼至關重要。在 UTF-8 解碼基準測試中，我們在低端 ARM 裝置上測得的效能得到了全面改進：英語文字提升至近 200%，中文文字提升至 400%。

發布外掛所需的新 pubspec.yaml 格式

當然，Flutter 不僅是 widget (在 Flutter 裡: Everything is a Widget)，它也是一個工具。今天這個版本中帶來的更新實在太多，無法一一提及。下面是一些亮點：

首先是一則宣告：如果你是 Flutter 外掛作者，發布外掛時將不再支援使用舊的 pubspec.yaml 格式。在使用舊格式檔案執行 pub publish 時會收到以下錯誤訊息：

△ 外掛發布時使用舊 pubspec 格式後收到的錯誤訊息

舊格式不能指定外掛支援的平台，自 Flutter 1.12 起已棄用。現在，發布新的外掛或更新外掛時需要使用 新的 pubspec.yaml 格式。

對於外掛的使用者，開發工具在當下和可預見的將來仍然能理解舊 pubspec 格式。在可預見的將來，pub.dev 上所有使用舊 pubspec.yaml 格式的既有外掛可繼續在 Flutter 應用中使用。

平台互操作的型別安全平台通道

為了回應外掛作者在使用者調研中的普遍需求，最近，我們一直以 外掛 和 Add-to-App （部分使用了 Flutter 的應用）為物件，探求如何才能讓 Flutter 與宿主平台之間的通訊更安全、更輕鬆。為了滿足這一需求，我們建立了指令行工具 Pigeon，使用 Dart 語法在平台通道上生成型別安全的訊息程式碼，無需新增其他執行時依賴項。你無需在平台通道上手動對應方法字串和序列化引數，就可以呼叫 Java/Objective-C/Kotlin/Swift 類方法，並透過直接呼叫 Dart 方法傳遞非原始型別資料物件（反之亦然）。

Pigeon 雖然處於預發布階段，但已經足夠成熟，我們已經將其用於 video_player 外掛。如果你有興趣測試 Pigeon 供自己使用，請參見更新的 平台通道文件 以及此 範例專案。
